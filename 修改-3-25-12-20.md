## 一、目标

- 用 **PaddleOCR-VL（AI Studio 任务接口）** 替代原课件/知识库 PDF 的本地文本提取链路（作为优先路径，失败时降级）。
- **PPT / PPTX** 先通过既有 **`ppt_to_pdf`** 转为 PDF，再参与解析。
- 在教师端、学生端 **保留原有页面与讲稿/问答流程**，仅在 PDF 预览上叠加 **分块悬停/点击**（与 `workspace-main` 方法说明一致）。
- 配置 **Paddle**、**DashScope 兼容接口 + Qwen** 等环境变量。

## 六、注意事项

1. **PPT/PPTX → PDF**：`ppt_to_pdf` 依次尝试 **WPS COM → Microsoft PowerPoint COM → LibreOffice（soffice --headless）**；转换失败时回退为 `python-pptx` 按页抽字，此时 **无 PDF 文件**，Paddle 不跑，**无 `paddlePages`**。
2. **密钥**：`.env` 中含 API Token，请勿提交到公开仓库；泄露后请在平台侧轮换。
3. 若 Paddle 未配置或调用失败，课件仍可完成解析（降级路径），但 **无 `paddlePages`**，前端不显示分块叠层。

## 七、补充（PPTX 与分块预览）

**原因说明**：分块叠层依赖 **PDF.js + Paddle 返回的 bbox**，且后端 **只对 PDF 调 Paddle**。若 PPTX **未转成 PDF**（本机无 WPS/PowerPoint/LibreOffice 或均失败），数据库里 `file_url` 仍为 `.pptx`，学生端走 **pptx-preview**，**不会出现**与 PDF 相同的分块交互。

**处理**：在服务器上安装 **LibreOffice**（并可将 `LIBREOFFICE_SOFFICE` 指向 `soffice.exe`），或确保 **WPS / Microsoft Office** 能完成转换；转换成功后 `file_url` 变为 `.pdf`，解析会写入 `paddlePages`，前端即可显示分块。
